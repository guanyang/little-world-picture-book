# Little World Picture Book - AI 辅助开发指南

此文档记录了本项目的核心架构设计、静态资源管线和开发规范，以便后续的 AI 编码助手能够快速理解上下文并保持代码和目录结构的一致性。

## 1. 项目概述与业务逻辑
- **业务定位**：面向低龄儿童的互动式启蒙认知绘本（卡片式 + 纸质绘本拟物风格）。
- **分类规模**：分为 **动物、植物、蔬菜、水果** 四个大类，每个大类包含正好 **20 个细分认知对象**（共计 80 个认知项目）。
- **发音功能**：调用浏览器原生的 `Speech Synthesis API`（朗读速率 `rate: 0.82`，音高 `pitch: 1.08`，语系 `zh-CN`）。

## 2. 静态资源管线与路径规范（核心）
项目采用“编译前自动切割拼图、自动进行 WebP 格式转换和压缩”的图像管线。请严格遵守以下路径规范以保持目录清爽：

- **开发源图（需提交 Git）**：
  - **拼板插画**：存放于 `assets/sheets/`。每个大分类有一个或多个大图画板（`.png` 格式）。
  - **大类场景图**：存放于 `assets/`，文件名为 `scene-*.jpg`。
- **本地切割缓存（已被 Git 忽略）**：
  - **缓存目录**：项目根目录下的 `.cache/`。
  - 裁剪脚本会在编译时将大画板图切成 80 个小图，并作为临时 JPEG 缓存写入 `.cache/`，以避免每次构建都重复裁剪，提升二次编译速度。
- **发布资源（需提交 Git）**：
  - **发布目录**：`public/media/optimized/`。
  - 临时 JPEG 最终会被转换为高压缩率的 WebP 文件并输出到这里。
  - **引用路径**：代码中通过静态路径 `/media/optimized/*.webp` 进行动态加载。
- **Favicon 规范**：
  - 存放于 `public/favicon.png`。
  - 已裁剪并调色板优化压缩至 `64x64` 像素（大小约 **1.86 KB**），切勿直接放置大图。

> [!IMPORTANT]
> **绝对禁止**将开发源图（如大画板 PNG）或临时切割文件直接放置在 `public/` 目录下，否则 Vite 会在打包时将它们原封不动复制到 `dist/`，导致生产部署包体积严重膨胀。

## 3. SEO 配置
- 模板文件 `index.html` 采用 `zh-CN` 语言属性。
- 内置了完整的 SEO meta 标签（Keywords, Description, Open Graph & Twitter Cards 社交分享标签），修改标题或添加新页面时需同步维护。

## 4. 常用开发命令
- **本地开发**：`npm run dev`（会自动先运行图片预处理脚本）。
- **生产构建**：`npm run build`（打包生成的 `dist/` 目录极其纯净，大小约 3.6 MB 左右，不含任何未压缩资源）。
- **图片预处理**：`npm run prepare-images`（手动执行 `scripts/prepare-images.mjs`，执行裁剪和压缩逻辑）。
