# Little World Picture Book - AI 辅助开发指南

此文档记录了本项目的核心架构设计、静态资源管线和开发规范，以便后续的 AI 编码助手能够快速理解上下文并保持代码和目录结构的一致性。

## 1. 项目概述与业务逻辑
- **业务定位**：面向低龄儿童的互动式启蒙认知绘本（卡片式 + 纸质绘本拟物风格）。
- **分类规模**：分为 **动物、植物、蔬菜、水果、交通工具、日常用品、天气与自然、身体与五官、职业与工具** 九个大类，每个大类包含正好 **20 个细分认知对象**（共计 180 个认知项目）。
- **发音功能**：调用浏览器原生的 `Speech Synthesis API`（朗读速率 `rate: 0.82`，音高 `pitch: 1.08`，语系 `zh-CN`）。

## 2. 静态资源管线与路径规范（核心）
项目采用“编译前自动切割拼图、自动进行 WebP 格式转换和压缩”的图像管线。请严格遵守以下路径规范以保持目录清爽：

- **开发源图（需提交 Git）**：
  - **拼板插画**：存放于 `assets/sheets/`。每个大分类有一个或多个大图画板（`.png` 格式）。
  - **大类场景图**：存放于 `assets/`，文件名为 `scene-*.jpg`。
- **本地切割缓存（已被 Git 忽略）**：
  - **缓存目录**：项目根目录下的 `.cache/`。
  - 裁剪脚本会在编译时将大画板图切成 180 个小图，并作为临时 JPEG 缓存写入 `.cache/`，以避免每次构建都重复裁剪，提升二次编译速度。
- **发布资源（需提交 Git）**：
  - **发布目录**：`public/media/optimized/`。
  - 临时 JPEG 最终会被转换为高压缩率的 WebP 文件并输出到这里。
  - **引用路径**：代码中通过静态路径 `/media/optimized/*.webp` 进行动态加载。
- **Favicon 规范**：
  - 存放于 `public/favicon.png`。
  - 已裁剪并调色板优化压缩至 `64x64` 像素（大小约 **1.86 KB**），切勿直接放置大图。

> [!IMPORTANT]
> **绝对禁止**将开发源图（如大画板 PNG）或临时切割文件直接放置在 `public/` 目录下，否则 Vite 会在打包时将它们原封不动复制到 `dist/`，导致生产部署包体积严重膨胀。

### 2.1 素材风格规范（新增或替换分类时必须遵守）

本项目的分类插画必须使用**严格真实感插画**方向，面向儿童但不做卡通化处理。新增或替换 `assets/scene-*.jpg`、`assets/sheets/*.png` 时，必须满足：

- **整体风格**：真实比例、真实材质、自然光影、清晰边缘，接近儿童百科/教育读物中的真实物体插画。
- **禁止风格**：不要卡通图、图标、贴纸、扁平矢量、玩具化造型、夸张比例、水彩绘本风、gouache/蜡笔/手绘涂鸦感。
- **场景图要求**：必须是同一主题下的真实感场景插画；不要内嵌文字、标签、logo、水印；左上区域尽量保留可覆盖标题的干净空间。
- **拼板素材要求**：必须严格按 `5 x 4` 网格（或既有 `5 x 1` 分段网格）组织；每格只放一个认知对象；对象居中、完整、不裁切；白色或浅色干净背景；对象顺序必须与 `src/content/bookData.js` 中该分类 `items` 顺序一致。
- **验收流程**：运行 `npm run prepare-images` 前，必须目检源图。如果素材板看起来像卡通、图标、贴纸或水彩绘本风，必须丢弃并重新生成/替换，不能把它作为最终源素材进入图片管线。
- **替换后验证**：替换源图后运行 `npm run prepare-images`、`npm test`、`npm run build`，并抽查 `public/media/optimized/` 中对应 WebP 是否保持真实感插画风格。

## 3. SEO 配置
- 模板文件 `index.html` 采用 `zh-CN` 语言属性。
- 内置了完整的 SEO meta 标签（Keywords, Description, Open Graph & Twitter Cards 社交分享标签），修改标题或添加新页面时需同步维护。

## 4. 常用开发命令
- **本地开发**：`npm run dev`（会自动先运行图片预处理脚本）。
- **生产构建**：`npm run build`（打包生成的 `dist/` 目录应保持纯净，不含任何未压缩源图）。
- **图片预处理**：`npm run prepare-images`（手动执行 `scripts/prepare-images.mjs`，执行裁剪和压缩逻辑）。
- **内容校验**：`npm test`（校验扩展分类数量、条目数量、slug 唯一性和图片路径规范）。
